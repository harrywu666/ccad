# Kimi k2p5 提示词优化建议报告

## 概述

本报告基于 Kimi k2p5 模型的技术特性，对室内装饰施工图AI自动审核系统的提示词进行全面分析并提出优化建议。

**当前提示词架构**：
- 5个核心阶段提示词（ai_prompt_service.py）
- 3个辅助Agent提示词（kimi_service.py内联）
- 温度设置：0.0-0.1（偏确定性）
- 输出格式：严格JSON

---

## 一、关键发现

### 1.1 整体质量评估

| 维度 | 评分 | 说明 |
|------|------|------|
| 结构化程度 | ⭐⭐⭐⭐⭐ | JSON格式约束清晰，字段定义完整 |
| 任务明确性 | ⭐⭐⭐⭐ | 大部分任务边界清晰，但部分语义分析缺乏深度 |
| 上下文利用 | ⭐⭐⭐ | 多图输入设计合理，但空间关系描述不足 |
| 容错设计 | ⭐⭐⭐⭐ | 缓存机制完善，但提示词缺乏异常处理指导 |

### 1.2 核心问题

1. **提示词过于"硬编码"** - 缺乏对k2p5推理能力的充分利用
2. **空间推理描述不足** - 施工图的轴网、坐标关系未显性化
3. **一致性约束较弱** - 跨阶段的数据关联缺乏约束
4. **错误处理缺失** - 没有指导模型如何处理模糊或矛盾信息

## 二、分阶段详细建议

### 2.1 目录识别 (catalog_recognition)

**当前提示词问题**：
```python
# 当前：过于简单，缺少对复杂表格结构的处理指导
default_user_prompt=(
    "你将收到2张图：第1张是目录表左侧放大图，第2张是全图。"
    "请以第1张为主提取所有目录条目，结合第2张纠正。"
    "只返回JSON数组，不要markdown，不要解释。"
)
```

**优化建议**：

1. **增加表格结构理解**：
```python
# 建议增加
"""
目录表通常具有以下结构特征：
- 表头行可能包含：序号、图号、图名、版本、日期、备注等列
- 图号列通常格式为：字母+数字（如A1-01、S-02）或纯数字（如01.02）
- 注意识别合并单元格和跨页续表
- 如果表格被截断，根据连续性推断缺失内容
"""
```

2. **增加异常处理指导**：
```python
"""
特殊情况处理：
- 如果某行数据不完整，优先填充图号和图名，版本和日期可为空字符串
- 如果识别到"续表"字样，说明表格跨页，需要与前页内容关联
- 如果图号格式不统一（如既有A1-01又有01.02），全部保留原始格式
- 如果一行有多个图号（如"A1-01~A1-05"），展开为多行独立条目
"""
```

3. **增强置信度输出**：
```python
# 建议输出格式增加置信度和原因
{
  "图号": "A1-01",
  "图名": "平面布置图",
  "置信度": 0.95,
  "识别依据": "标题栏清晰可读",
  "异常标记": null
}
```

### 2.2 单页图纸识别 (sheet_recognition)

**当前提示词问题**：
- 三图输入设计好，但缺乏空间对应关系说明
- 没有处理不同图纸类型的差异化指导
- 缺少对模糊/遮挡情况的处理

**优化建议**：

1. **增加空间对应关系说明**：
```python
"""
三张裁剪图的空间关系：
- 左侧图：图纸左半部分，通常包含主视图和左侧标注
- 右侧图：图纸右半部分，通常包含右侧标注和图签（标题栏）
- 下方图：图纸底部，通常包含图签、比例、设计信息

识别优先级：
1. 首先在右侧图和下方图查找图签（右下角区域）
2. 图签中通常包含：图号、图名、比例、日期
3. 如果图签模糊，结合左侧图的整体内容推断图纸类型

图纸类型识别线索：
- 平面图：包含房间布局、家具布置、多尺寸标注
- 立面图：显示墙面竖向设计、高度标注
- 详图/大样：局部放大，标注密集
"""
```

2. **增加模糊处理策略**：
```python
"""
识别困难时的处理策略：
1. 如果图签文字模糊，根据图纸整体布局推断：
   - 图纸编号通常遵循项目编号规则（如A1-01到A1-05是连续平面）
   - 图名根据图纸类型关键词推断（平面/立面/详图）
2. 如果置信度低于0.7，在"依据"字段说明不确定性来源
3. 如果完全无法识别，图号和图名返回空字符串，置信度0.0
"""
```

### 2.3 总控任务规划 (master_task_planner)

**当前提示词问题**：
- 规则约束过多，限制了模型的推理能力
- 缺乏对图纸依赖关系的深层理解
- 任务优先级逻辑过于简单

**优化建议**：

1. **重构提示词结构 - 从"规则约束"转向"推理引导"**：
```python
# 当前：硬性约束
"""
必须遵守：
1) 只允许 task_type 为 index / dimension / material。
2) 所有图号必须严格来自输入 contexts.sheet_no，不能编造。
3) dimension/material 任务必须严格来自输入 edges 的 source->target...
"""

# 建议：推理引导
"""
你是CAD审核系统的总控规划师，任务是根据图纸关系和上下文规划最优审核路径。

规划原则：
1. **完整性原则**：确保每张有索引的图纸都被检查，每条索引边都被验证
2. **依赖顺序**：
   - 先执行索引核对（index），确认图纸引用关系正确
   - 再执行尺寸核对（dimension），基于索引边比对平立面尺寸
   - 最后执行材料核对（material），检查材料一致性
3. **优先级策略**：
   - 平面图（A1开头/含"平面"关键词）优先级最高（1-2级）
   - 索引多的图纸优先检查
   - 被引用次数多的图纸优先作为source

图纸关系分析：
- 输入的 edges 表示索引起引关系（source图纸指向target图纸）
- 每个 edge 可能涉及多个索引编号
- 根据 edge_mention_count 判断关系强度，高频引用优先处理

任务类型定义：
- index：单图检查，验证图纸内所有索引的合法性和闭合性
- dimension：双图比对，对比source和target图纸的尺寸一致性
- material：双图比对，检查source和target图纸的材料标注一致性
"""
```

2. **增加冲突检测指导**：
```python
"""
冲突检测与处理：
1. 如果发现图纸A既指向B又指向C，但B和C是同一空间的不同详图，标记为potential_conflict
2. 如果发现闭环引用（A->B->C->A），这是正常的设计模式，不需要特别处理
3. 如果发现孤立图纸（没有任何edge关联），标记为isolated_sheet
"""
```

### 2.4 尺寸单图语义分析 (dimension_single_sheet)

**当前提示词问题**：
- "先定位，再理解语义"过于抽象
- 缺少对尺寸类型的分类指导
- 缺乏对坐标系和比例的说明

**优化建议**：

1. **增加空间推理框架**：
```python
"""
语义解析步骤（严格按此顺序执行）：

步骤1：图纸整体理解
- 查看全图（图1）识别图纸类型和轴线网格
- 轴线通常标注为：横向①②③...、纵向A B C...
- 使用棋盘格（A1-X17）定位主要构件

步骤2：尺寸分类
将尺寸标注分为以下类型：
- 总尺寸：房间或空间的总长度/宽度（通常最大）
- 分段尺寸：构成总尺寸的各部分尺寸
- 定位尺寸：构件相对于轴线的定位距离
- 细部尺寸：局部详图的精确尺寸

步骤3：语义关联
- 每个尺寸需要关联到：位置（网格区域）、构件（墙体/门窗/家具）、类型
- 使用grid字段记录尺寸在棋盘格中的大致位置（如"D5-E6"）
- 使用component字段描述构件类型（如"主卧东墙"、"客厅南向窗户"）

步骤4：跨图准备
- 识别可能与其他图纸（如立面图、详图）相关的尺寸
- 在evidence字段记录关联线索（如"此墙在A2-03有对应立面"）
"""
```

2. **增强置信度评估**：
```python
# 建议输出增加不确定性标记
{
  "id": "dim_001",
  "semantic": "主卧东侧墙体净宽",
  "location_desc": "主卧B-C轴/①-②轴区域东墙",
  "dim_type": "分段尺寸",
  "value": 3600,
  "grid": "C5-D6",
  "component": "墙体",
  "confidence": 0.92,
  "uncertainty_flags": ["文字略有模糊"],
  "evidence": {
    "grid": "C5-D6",
    "why": "标注位于主卧区域，与B-C轴平行，对应东墙位置"
  }
}
```

### 2.5 尺寸双图对比 (dimension_pair_compare)

**当前提示词问题**：
- "先定位 -> 再配对 -> 再核对"过于简单
- 缺少对差异容忍度的指导
- 缺乏对边缘情况的处理

**优化建议**：

1. **增加对比策略框架**：
```python
"""
双图对比策略（按此优先级执行）：

配对优先级（从高到低）：
1. **网格坐标匹配**：两图中位于相同或相邻网格区域的尺寸优先配对
2. **构件语义匹配**：semantic字段描述相同构件的尺寸配对
3. **数值接近匹配**：在相同网格区域内，数值接近的尺寸配对
4. **索引线索匹配**：如果图纸A通过索引①指向图纸B，则A中索引附近的尺寸与B中①号详图区域尺寸配对

差异判断标准：
- 精确匹配：数值完全相同（差值=0）→ 无问题
- 工程误差：差值≤5mm → 标记为tolerance（施工误差允许范围）
- 可疑差异：5mm < 差值 ≤ 20mm → 标记为warning（需要确认）
- 明显错误：差值 > 20mm → 标记为error（必须整改）

特殊情况处理：
1. **总尺寸与分段尺寸**：如果A图总尺寸 = B图分段尺寸之和，即使单个分段不一致，也标记为"分段标注差异"
2. **单位换算**：注意mm和m的混淆（如2400 vs 2.4）
3. **标注方向**：平面图水平尺寸 vs 立面图竖向尺寸，需要空间位置对应而非单纯数值对比
4. **多对一情况**：一个平面图区域可能对应多个立面详图，需要分别配对
"""
```

2. **增加证据链要求**：
```python
"""
每个问题必须提供完整的证据链：
{
  "位置描述": "主卧东侧墙体净宽",
  "A图号": "A1-01", 
  "B图号": "A2-03",
  "A值": 3600,
  "B值": 3400,
  "差值": 200,
  "source_grid": "C5",
  "target_grid": "详图①区域",
  "source_dim_id": "dim_042",
  "target_dim_id": "dim_128",
  "index_hint": "A1-01通过索引①指向A2-03",
  "confidence": 0.88,
  "description": "平面图A1-01中主卧东侧墙体净宽标注为3600mm，对应立面图A2-03详图①区域同位置标注为3400mm，差值200mm，建议核实",
  "evidence": {
    "source_sheet_no": "A1-01",
    "target_sheet_no": "A2-03",
    "source_grid": "C5-C6",
    "target_grid": "详图①左侧",
    "source_dim_id": "dim_042",
    "target_dim_id": "dim_128",
    "confidence": 0.88,
    "why": "两图均标注主卧东墙，网格位置对应（C5），但通过索引关联后数值差异200mm"
  }
}
"""
```

## 四、具体代码改进建议

### 4.1 kimi_service.py 汇总Agent优化

**当前**：
```python
system_prompt="你是施工图汇总Agent，只输出JSON。"
```

**建议**：
```python
system_prompt="""
你是施工图汇总Agent，任务是对多页图纸识别结果进行统一校正。

校正原则：
1. **一致性检查**：检查同一项目的图号是否遵循统一编号规则
2. **连续性检查**：检查页码顺序是否合理（如A1-01后面应该是A1-02）
3. **命名规范化**：统一图名表述（如"平面布置图"和"平面图"统一为"平面布置图"）

输出要求：
- 保留原始置信度较高的结果
- 对低置信度结果进行智能修正
- 在"理由"字段说明修正依据
- 返回完整数组，不要遗漏任何页
"""
```

### 4.2 温度参数调优建议

当前所有调用使用temperature=0.0（完全确定性），建议：

| 任务类型 | 当前温度 | 建议温度 | 理由 |
|---------|---------|---------|------|
| 目录识别 | 0.0 | 0.0-0.1 | 结构化数据，需要稳定 |
| 图纸识别 | 0.0 | 0.0-0.1 | 结构化数据，需要稳定 |
| 任务规划 | 0.0 | 0.2-0.3 | 需要一定灵活性优化路径 |
| 单图语义 | 0.0 | 0.1-0.2 | 语义理解需要适度推理 |
| 双图对比 | 0.0 | 0.0 | 精确对比，避免幻觉 |

## 五、优先级建议

### 高优先级（立即实施）

1. **dimension_single_sheet**：增加空间推理框架和尺寸分类指导
2. **dimension_pair_compare**：增加差异容忍度和证据链要求
3. **错误处理**：所有提示词增加"特殊情况处理"章节

### 中优先级（1-2周内）

1. **catalog_recognition**：增加表格结构理解和异常处理
2. **sheet_recognition**：增加空间对应关系和图纸类型识别
3. **汇总Agent**：增加校正原则和一致性检查

### 低优先级（持续优化）

1. 提示词版本管理系统
2. A/B测试框架

## 六、验证建议

在实施优化后，建议进行以下验证：

1. **回归测试**：使用已有测试集验证准确率不下降
2. **置信度监控**：追踪模型返回的置信度分布变化
3. **边缘案例测试**：专门测试模糊、遮挡、非常规布局图纸

---

## 结语

当前系统的提示词设计已经达到可用水平，但在**空间推理**、**异常处理**和**跨阶段一致性**方面还有提升空间。建议优先优化尺寸审核相关的提示词，因为这是系统的核心价值所在。

k2p5模型在施工图理解方面有很强的潜力，关键在于如何通过提示词引导模型运用其空间推理和逻辑分析能力。从"规则约束"转向"推理引导"是核心优化方向。
