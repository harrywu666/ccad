## 三、数据流转详解

### 3.1 目录识别流程

```
用户上传目录PNG
    ↓
后端接收文件，保存到本地临时目录
    ↓
调用Kimi API，发送PNG + Prompt
Prompt内容：
  "这是一张室内装饰施工图的图纸目录页，
   请提取所有图纸条目，严格按以下JSON格式返回，不要其他内容：
   [{'图号':'A1-01','图名':'平面布置图','版本':'A','日期':'2026.01'}]
   如果某个字段识别不到，用空字符串代替"
    ↓
解析Kimi返回的JSON
    ↓
存入数据库 catalog表
    ↓
返回前端展示，等待用户校对
    ↓
用户确认后，catalog状态标记为 locked
```

### 3.2 PDF转PNG流式处理流程

```
用户上传PDF
    ↓
后端接收PDF，保存到本地
检查该项目是否已有PDF数据：
  → 有：清除旧PNG文件和drawings记录，开始替换流程
  → 无：直接开始处理
    ↓
PyMuPDF打开PDF，获取总页数
    ↓
逐页处理（流式）：
  for 每一页:
    1. 渲染为300DPI的PNG（bytes，不存磁盘）
    2. 发给Kimi识别图名图号
    3. 与目录匹配
    4. 存入数据库drawings表
    5. 通过WebSocket推送进度到前端
    ↓
全部处理完成，前端展示匹配结果
更新项目缓存版本号（cache_version+1）
```

### 3.3 DWG数据提取流程

> **核心逻辑：一个DWG文件包含一个Model（模型空间）和多个Layout（布局），
> 每个布局对应一张施工图纸。必须按布局拆分，每个布局导出一个独立JSON。
> 例如：上传20个DWG文件，共含44个布局 → 导出44个JSON，匹配44张图纸。**
>
> **技术方案：ODA File Converter（DWG→DXF） + ezdxf（Python解析DXF）**
> **无需安装AutoCAD，Mac/Windows均可运行。**

```
用户批量上传DWG文件
    ↓
后端接收，保存到临时目录
    ↓
调用 ODA File Converter（命令行）：
  输入目录：所有上传的DWG文件
  输出目录：临时DXF目录
  命令：ODAFileConverter input/ output/ ACAD2018 DXF 0 1
  ↓
  ODA批量将每个DWG转换为DXF文件（秒级完成）
    ↓
Python用ezdxf解析每个DXF文件：
  对每个DXF：
    doc = ezdxf.readfile("xxx.dxf")
    遍历所有Layout（跳过"Model"模型空间）
    对每个Layout（= 一张图纸）：
      1. 读取所有VIEWPORT实体
         → 视口中心点、模型空间中心点、视口高度、激活图层
         → 计算 model_range（视口覆盖的模型空间范围）
      2. 筛选可见图层（冻结/关闭的图层不提取）
      3. 提取DIMENSION实体
         → dim.dxf.actual_measurement（精确值）
         → defpoint/defpoint2坐标
         → 所在图层
      4. 提取INSERT实体（图块）
         → 图块名含"索引"/"INDEX" → 索引符号
         → 图块名含图签/标题栏关键词 → 图签信息（图号图名）
         → 提取所有ATTRIB属性值
      5. 提取MLEADER/LEADER实体
         → 引线文字内容（材料标注）
         → 引线箭头坐标
      6. 提取TABLE实体（材料总表）
         → 无TABLE则尝试识别TEXT实体拼合的表格
      7. 生成该布局的JSON文件
         文件名：{dwg文件名}_{布局名}.json
    ↓
  一个DXF可生成多个JSON（有几个布局生成几个）
    ↓
检查各布局JSON是否已有历史版本：
  有 → is_latest改为0（保留历史）
  无 → 直接写入
    ↓
新JSON记录写入json_data表（is_latest=1，version递增）
    ↓
更新项目缓存版本号
    ↓
返回前端：展示每个DWG拆分出的布局列表和数据详情
```

### 3.4 三线匹配汇合

```
以catalog为基准，遍历每个目录条目
找到对应的drawings记录（通过图号）
找到对应的json_data记录（通过图号）
三者绑定

状态：
  三者都有 → status: ready
  缺PNG    → status: missing_png
  缺JSON   → status: missing_json
  都缺     → status: missing_all
```

### 3.5 审核核对流程

```
遍历所有status为ready的图纸

第一步：索引核对
  从所有JSON中提取索引数据
  构建全局索引映射表
  Python逻辑检测断链
  结果存入audit_results表（type: index）

第二步：尺寸核对
  基于索引关系，找到平面图对应的立面图
  对每对平立面：
    从JSON获取精确尺寸数值
    裁出对应区域的局部PNG
    调用Kimi（全图缩略+局部高清+JSON数值）
    结果存入audit_results表（type: dimension）

第三步：材料核对
  从JSON提取材料表和材料标注
  Python精确比对 + Kimi模糊匹配
  结果存入audit_results表（type: material）

生成最终报告
```

## 四、数据库结构

### 4.1 projects表（项目）
```sql
CREATE TABLE projects (
    id            TEXT PRIMARY KEY,
    name          TEXT NOT NULL,
    category      TEXT,      -- 项目分类，如「住宅」「商业」「办公」「酒店」
    tags          TEXT,      -- 标签JSON数组，如 '["上海","2026年","精装修"]'
    description   TEXT,      -- 项目备注
    cache_version INTEGER DEFAULT 0,  -- 缓存版本号，数据更新时+1
    created_at    DATETIME,
    status        TEXT,      -- new/catalog_locked/matching/ready/auditing/done
    updated_at    DATETIME
);
```

### 4.2 project_categories表（分类）
```sql
CREATE TABLE project_categories (
    id         TEXT PRIMARY KEY,
    name       TEXT NOT NULL UNIQUE,  -- 分类名称，如「住宅」
    color      TEXT,                  -- 分类颜色标识，如 "#3B82F6"
    sort_order INTEGER DEFAULT 0,
    created_at DATETIME
);

-- 预置默认分类
INSERT INTO project_categories (id, name, color) VALUES
  ('cat_1', '住宅', '#3B82F6'),
  ('cat_2', '商业', '#F59E0B'),
  ('cat_3', '办公', '#10B981'),
  ('cat_4', '酒店', '#8B5CF6'),
  ('cat_5', '其他', '#6B7280');
```

### 4.3 catalog表（图纸目录）
```sql
CREATE TABLE catalog (
    id          TEXT PRIMARY KEY,
    project_id  TEXT,
    sheet_no    TEXT,   -- 图号，如 A1-01
    sheet_name  TEXT,   -- 图名，如 平面布置图
    version     TEXT,   -- 版本
    date        TEXT,   -- 日期
    status      TEXT,   -- pending/locked
    sort_order  INTEGER DEFAULT 0,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);
```

### 4.4 drawings表（图纸PNG）
```sql
CREATE TABLE drawings (
    id              TEXT PRIMARY KEY,
    project_id      TEXT,
    catalog_id      TEXT,
    sheet_no        TEXT,
    sheet_name      TEXT,
    png_path        TEXT,
    page_index      INTEGER,
    data_version    INTEGER DEFAULT 1,  -- 数据版本号
    replaced_at     DATETIME,           -- 被替换的时间（有值=已替换）
    status          TEXT,   -- matched/unmatched
    FOREIGN KEY (project_id) REFERENCES projects(id)
);
```

### 4.5 json_data表（DWG提取数据）
```sql
CREATE TABLE json_data (
    id              TEXT PRIMARY KEY,
    project_id      TEXT,
    catalog_id      TEXT,
    sheet_no        TEXT,
    json_path       TEXT,
    data_version    INTEGER DEFAULT 1,  -- 数据版本号，重新上传+1
    is_latest       INTEGER DEFAULT 1,  -- 1=当前版本，0=历史版本
    summary         TEXT,
    created_at      DATETIME,
    status          TEXT,   -- pending/matched
    FOREIGN KEY (project_id) REFERENCES projects(id)
);
```

### 4.6 audit_results表（审核结果）
```sql
CREATE TABLE audit_results (
    id              TEXT PRIMARY KEY,
    project_id      TEXT,
    audit_version   INTEGER DEFAULT 1,  -- 审核版本号，重新审核+1
    type            TEXT,   -- index/dimension/material
    severity        TEXT,   -- error/warning/info
    sheet_no_a      TEXT,
    sheet_no_b      TEXT,
    location        TEXT,
    value_a         TEXT,
    value_b         TEXT,
    description     TEXT,   -- AI生成的自然语言描述
    created_at      DATETIME,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);
```

## 五、数据替换与缓存更新机制

### 5.1 数据替换策略

**替换目录：**
```
用户重新上传目录PNG
→ 原catalog记录全部删除
→ 重新识别，写入新记录
→ 已关联的drawings/json_data的catalog_id置空（需要重新匹配）
→ project.cache_version + 1
→ 通知前端刷新
```

**替换PDF（图纸PNG）：**
```
用户重新上传PDF
→ 原drawings记录标记replaced_at为当前时间（软删除，保留历史）
→ 重新处理PDF，写入新drawings记录（data_version = 旧版本+1）
→ 重新执行与catalog的匹配
→ project.cache_version + 1
→ 通知前端刷新
```

**替换DWG（JSON数据）：**
```
用户重新上传某个DWG文件
→ 对应的json_data记录 is_latest 改为 0（历史版本保留）
→ 写入新json_data记录（data_version = 旧版本+1，is_latest = 1）
→ project.cache_version + 1
→ 通知前端刷新
→ 提示用户：「数据已更新，建议重新审核」
```

### 5.2 前端缓存策略

前端在每次请求时带上本地缓存的cache_version：
```javascript
// 请求示例
GET /api/projects/{id}?client_version=3

// 后端响应
{
  "cache_version": 5,        // 服务器最新版本
  "client_version": 3,       // 客户端版本
  "needs_refresh": true,     // 需要刷新
  "data": { ... }
}
```

前端收到 needs_refresh: true 时，清除本地缓存，重新拉取所有数据。

### 5.3 重新审核机制

当数据被替换后，系统提示用户是否重新审核：
```
检测到以下数据已更新：
  - A1-01 平面布置图（DWG重新上传，v1→v2）
  - A2-03 立面图（DWG重新上传，v1→v2）

上次审核时间：2026.02.01 14:35
建议重新审核以获取最新结果。

[查看上次结果]  [重新审核]
```

## 六、JSON数据结构（CAD插件输出）

### 核心规则：一个布局 = 一个JSON文件

**不是一个DWG一个JSON，而是一个布局一个JSON。**

```
某工程.dwg（含3个布局）
  ├── 布局：平面布置图  → A1_01_平面布置图.json
  ├── 布局：地面铺装图  → A1_01_地面铺装图.json
  └── 布局：天花布置图  → A1_01_天花布置图.json

另一工程.dwg（含2个布局）
  ├── 布局：A2-01立面图 → A2_01_A2-01立面图.json
  └── 布局：A2-02立面图 → A2_01_A2-02立面图.json
```

每个JSON只描述一个布局（一张图纸）的数据，格式如下：

```json
{
  "source_dwg":   "A1_01.dwg",
  "layout_name":  "平面布置图",
  "sheet_no":     "A1-01",
  "extracted_at": "2026-02-01 14:30",
  "data_version": 1,
  "scale":        "1:50",
  "model_range": {
    "min": [0.0, 0.0],
    "max": [15000.0, 10000.0]
  },
  "dimensions": [
    {
      "id":           "dim_001",
      "value":        2400,
      "display_text": "2400",
      "layer":        "S-DIMS",
      "source":       "model_space",
      "defpoint":     [1200.0, 3500.0],
      "defpoint2":    [3600.0, 3500.0],
      "text_position":[2400.0, 3600.0]
    }
  ],
  "indexes": [
    {
      "id":           "idx_001",
      "index_no":     "①",
      "target_sheet": "A2-01",
      "source":       "layout_space",
      "position":     [5600.0, 2300.0]
    }
  ],
  "materials": [
    {
      "id":       "mat_001",
      "code":     "F1",
      "name":     "餐桌 DINING TABLE",
      "position": [3200.0, 4100.0]
    }
  ],
  "material_table": [
    {"code": "F1", "name": "餐桌 DINING TABLE"},
    {"code": "F2", "name": "餐椅 DINING CHAIR"}
  ]
}
```

### 字段说明

| 字段 | 说明 |
|------|------|
| `source_dwg` | 来源DWG文件名 |
| `layout_name` | 布局名称（AutoCAD里的布局选项卡名） |
| `sheet_no` | 图号（从布局名解析，或Kimi识别图签后补充） |
| `model_range` | 视口对应的模型空间范围（用于坐标换算） |
| `dimensions` | 该布局可见的所有尺寸标注（已统一换算为模型坐标） |
| `indexes` | 该布局中的所有索引符号 |
| `materials` | 该布局中的材料引线标注 |
| `material_table` | 该布局中的材料总表（如无则为空数组） |

## 八、本地部署说明

### 8.1 用户电脑要求
- macOS 12+ 或 Windows 10/11（两者均支持）
- Python 3.10+
- Node.js 18+
- ODA File Converter（免费下载，见8.4节安装说明）

- **不再需要安装 AutoCAD**

**后端（端口7000）：**
```bash
cd cad-review-backend
source venv/bin/activate
uvicorn main:app --reload --port 7000
```

**前端（端口7001）：**
```bash
cd cad-review-frontend
npm start -- --port 7001
```

前端API请求地址统一配置为 `http://localhost:7000`

### 8.3 ODA File Converter 安装说明

ODA File Converter 是由 Open Design Alliance（DWG格式标准制定方）发布的免费工具，
专门用于DWG/DXF格式互转，无需AutoCAD授权。

**下载地址：** https://www.opendesign.com/guestfiles/oda_file_converter

**Mac安装：**
```bash
# 下载 .dmg 后安装，默认路径：
/Applications/ODAFileConverter.app/Contents/MacOS/ODAFileConverter
```

**Windows安装：**
```bash
# 下载 .exe 后安装，默认路径：
C:\Program Files\ODA\ODAFileConverter\ODAFileConverter.exe
```

**命令行调用格式：**
```bash
ODAFileConverter <输入目录> <输出目录> <版本> <格式> <是否递归> <是否审计>

# 示例：将 input/ 目录下所有DWG批量转为DXF，输出到 output/
ODAFileConverter /tmp/dwg_in /tmp/dxf_out ACAD2018 DXF 0 1

# 参数说明：
# ACAD2018  → 目标版本（兼容性最好，建议固定此值）
# DXF       → 输出格式
# 0         → 不递归子目录
# 1         → 启用审计（自动修复文件错误）
```

**Python中调用：**
```python
import subprocess, platform, shutil

def get_oda_path() -> str:
    if platform.system() == "Darwin":  # Mac
        return "/Applications/ODAFileConverter.app/Contents/MacOS/ODAFileConverter"
    else:  # Windows
        return r"C:\Program Files\ODA\ODAFileConverter\ODAFileConverter.exe"

def dwg_batch_to_dxf(input_dir: str, output_dir: str) -> list[str]:
    """批量将input_dir中所有DWG转为DXF，返回生成的DXF文件路径列表"""
    oda = get_oda_path()
    if not shutil.which(oda) and not os.path.exists(oda):
        raise RuntimeError("未找到ODA File Converter，请先安装")
    subprocess.run(
        [oda, input_dir, output_dir, "ACAD2018", "DXF", "0", "1"],
        check=True, capture_output=True
    )
    return [str(p) for p in Path(output_dir).glob("*.dxf")]
```

## 九、Kimi API 接入规范

> ⚠️ **重要：本项目使用 Kimi Code 订阅方案，不是 Moonshot 开放平台，两者 API 端点和 Key 格式完全不同，不可混用。**

### 9.1 核心配置

```python
KIMI_API_BASE  = "https://api.kimi.com/coding/v1"   # ❌ 不是 api.moonshot.cn
KIMI_MODEL     = "k2p5"                              # ❌ 不是 kimi-k2.5 / kimi-latest
KIMI_UA        = "claude-code/1.0"                  # ❌ 不能省略，否则 403
```

| 参数 | 值 | 说明 |
|------|----|------|
| `model` | `k2p5` | 性能模式模型，Kimi Code 订阅专用名 |
| `temperature` | `0.1` | 低温确保结构化输出稳定 |
| `max_tokens` | `65536` | 设最大值，防止长JSON被截断；订阅制不按token计费 |
| `User-Agent` | `claude-code/1.0` | 必须伪装为 Coding Agent，否则返回 403 |

### 9.2 为什么必须伪装 User-Agent

Kimi Code API 会校验请求来源，只允许特定 Coding Agent 调用：
```
403 Kimi For Coding is currently only available for Coding Agents
such as Kimi CLI, Claude Code, Roo Code, Kilo Code, etc.
```
已验证只有 `claude-code/1.0` 可用，其他均返回 403。

### 9.3 标准封装（Anthropic 格式，推荐）

所有 Kimi 调用统一使用 **Anthropic 格式 + httpx**，不使用 OpenAI SDK（SDK 无法可靠覆盖 User-Agent）。

`services/kimi_service.py` 完整代码：

```python
import os, re, json, base64, logging
import httpx

logger = logging.getLogger(__name__)

KIMI_API_BASE = "https://api.kimi.com/coding/v1"
KIMI_MODEL    = "k2p5"
KIMI_UA       = "claude-code/1.0"


def _headers() -> dict:
    key = os.getenv("KIMI_CODE_API_KEY", "")
    if not key:
        raise ValueError("未设置 KIMI_CODE_API_KEY 环境变量")
    return {
        "Content-Type":      "application/json",
        "x-api-key":         key,              # Anthropic 格式用 x-api-key
        "anthropic-version": "2023-06-01",
        "User-Agent":        KIMI_UA,          # 关键！不能省略
    }


def _mime(data: bytes) -> str:
    """根据文件头自动判断图片格式"""
    b = base64.b64encode(data[:4]).decode()
    if b.startswith("iVBOR"): return "image/png"
    if b.startswith("R0lGO"): return "image/gif"
    if b.startswith("UklGR"): return "image/webp"
    return "image/jpeg"


def _parse_json(text: str) -> dict | list:
    """健壮的 JSON 提取：直接解析 → 剥离代码块 → 暴力搜索"""
    t = text.strip()
    try:
        return json.loads(t)
    except json.JSONDecodeError:
        pass
    m = re.search(r"```(?:json)?\s*\n?([\s\S]*?)\n?\s*```", t)
    if m:
        try:
            return json.loads(m.group(1).strip())
        except json.JSONDecodeError:
            pass
    # 暴力搜索第一个 { 到最后一个 }
    a, b = t.find("{"), t.rfind("}")
    if a != -1 and b > a:
        try:
            return json.loads(t[a:b+1])
        except json.JSONDecodeError:
            pass
    # 暴力搜索数组 [ ... ]
    a, b = t.find("["), t.rfind("]")
    if a != -1 and b > a:
        try:
            return json.loads(t[a:b+1])
        except json.JSONDecodeError:
            pass
    raise ValueError(f"无法提取 JSON，原文前500字：{text[:500]}")


async def call_kimi(
    system_prompt: str,
    user_prompt: str,
    images: list[bytes] | None = None,
    temperature: float = 0.1,
) -> dict | list:
    """
    统一 Kimi 调用入口（支持纯文本和多图混合）
    images: 图片 bytes 列表，None 或空列表表示纯文本请求
    返回解析后的 JSON 对象或列表
    """
    content: list[dict] = []

    # 先放图片（可多张）
    for img in (images or []):
        content.append({
            "type": "image",
            "source": {
                "type":       "base64",
                "media_type": _mime(img),
                "data":       base64.b64encode(img).decode(),
            },
        })

    # 再放文字
    content.append({"type": "text", "text": user_prompt})

    payload = {
        "model":       KIMI_MODEL,
        "system":      system_prompt,
        "messages":    [{"role": "user", "content": content}],
        "temperature": temperature,
        "max_tokens":  65536,          # 设最大，防止 JSON 截断
    }

    async with httpx.AsyncClient(timeout=300.0) as client:
        resp = await client.post(
            f"{KIMI_API_BASE}/messages",
            headers=_headers(),
            json=payload,
        )

    if resp.status_code != 200:
        raise RuntimeError(f"Kimi API 失败 ({resp.status_code}): {resp.text[:300]}")

    data = resp.json()
    raw = "".join(
        b.get("text", "")
        for b in data.get("content", [])
        if b.get("type") == "text"
    )
    logger.info("Kimi 响应长度: %d 字符", len(raw))
    return _parse_json(raw)


async def test_kimi_connection() -> dict:
    """启动时调用，验证 API Key 和网络连通性"""
    try:
        result = await call_kimi(
            system_prompt="You are a test assistant.",
            user_prompt='Return exactly: {"status":"ok"}',
        )
        return {"ok": True, "result": result}
    except Exception as e:
        return {"ok": False, "error": str(e)}
```

### 9.5 常见坑

| 坑 | 现象 | 解决 |
|----|------|------|
| 用 `api.moonshot.cn` | 401 Invalid Authentication | 换 `api.kimi.com/coding/v1` |
| 缺少 User-Agent | 403 only available for Coding Agents | 加 `User-Agent: claude-code/1.0` |
| 用 OpenAI SDK | User-Agent 被 SDK 覆盖，403 | 改用 httpx 直接调用 |
| `max_tokens` 过小 | JSON 尾部被截断，解析失败 | 设为 65536 |
| 模型名 `kimi-k2.5` | 报错或识别不了 | 改用 `k2p5` |
| AI 返回 JSON 包裹在代码块 | `json.loads` 失败 | 用 `_parse_json()` 多层提取 |
