# 文档三：开发计划
## 室内装饰施工图 AI 自动审核系统

**版本：V1.1 | 日期：2026年2月**
**更新：新增shadcn/ui、项目分类管理、数据替换机制，端口改为后端7000/前端7001**

## 一、开发总览

### 1.1 阶段划分

| 阶段 | 名称 | 预估周期 | 核心交付物 |
|------|------|---------|-----------|
| 第零阶段 | 环境搭建与准备 | 1周 | 开发环境全部就绪 |
| 第一阶段 | 项目骨架搭建 | 1周 | 前后端跑通，数据库建好 |
| 第二阶段 | 项目管理模块 | 1～2周 | 项目列表、分类、数据留存 |
| 第三阶段 | 目录识别模块 | 1～2周 | 目录上传、识别、校对、锁定 |
| 第四阶段 | PDF转PNG模块 | 1～2周 | PDF流式处理，图名图号匹配 |
| 第五阶段 | CAD插件模块 | 2～3周 | DWG数据提取，生成JSON |
| 第六阶段 | 三线匹配模块 | 1周 | 目录/PNG/JSON完整匹配 |
| 第七阶段 | 审核引擎模块 | 3～4周 | 三步审核逻辑全部跑通 |
| 第八阶段 | 报告生成模块 | 1～2周 | PDF和Excel报告可下载 |
| 第九阶段 | 界面完善与测试 | 2周 | 完整可用的产品 |

**总预估周期：3～4个月**

## 七、第五阶段：CAD插件模块

> ⚠️ **此阶段必须在Windows环境下开发和测试**

### 任务5-1：AutoLISP基础框架 + 布局遍历

> **核心设计原则：每个布局 = 一张图纸 = 一个JSON文件。
> 跳过Model空间，只处理Layout布局选项卡。**

打开Trae（Windows环境），输入：
```
帮我写AutoLISP脚本 extract_data.lsp，功能是：
1. 定义命令 EXTRACTDATA
2. 遍历当前DWG文件中所有布局（Layout），跳过名为"Model"的模型空间
3. 对每个布局，获取布局名称
4. 打印所有布局名称（用于测试）
5. 输出示例：「找到 3 个布局：平面布置图 / 地面铺装图 / 天花布置图」
```

### 任务5-2：单布局数据提取函数

打开Trae，输入：
```
帮我在 extract_data.lsp 中写函数 extract-layout-data(layout-name)：
进入指定布局后，提取以下数据：

1. 视口信息（VIEWPORT实体）：
   - 视口在布局空间的中心点（DXF组码10）
   - 视口对应模型空间的中心点（DXF组码17）
   - 视口缩放比例（DXF组码45）
   - 视口激活的图层列表（DXF组码301）
   记录为 viewports 列表，用于后续坐标换算

2. 模型空间标注（在视口范围内）：
   - 对每个视口，筛选视口model_range范围内的DIMENSION实体
   - 用 getpropertyvalue 获取 Measurement 精确值（不用文字显示值）
   - 提取 defpoint、defpoint2 坐标（模型空间坐标，不需换算）
   - 标记 source: "model_space"

3. 布局空间标注（直接在布局纸上的）：
   - 筛选布局空间中的DIMENSION实体
   - 将布局坐标换算为模型坐标：
     model_pt = model_center + (layout_pt - viewport_center) / scale
   - 标记 source: "layout_space"

4. 索引图块（INSERT实体，图块名含"索引"或"INDEX"）：
   提取插入点坐标和所有属性值（ATTRIB子实体）

5. 材料引线（MLEADER/LEADER实体）：
   提取文字内容和引线起点

6. 材料表（TABLE实体，无则尝试按行列重建TEXT实体）

返回以上数据的完整列表结构
```

```
   c. 新记录写入json_data表（is_latest=1，data_version递增）
   d. JSON文件保存到 ~/cad-review/projects/{id}/jsons/
      文件名：{布局名}_v{version}.json
4. increment_cache_version()
5. WebSocket推送完成消息：
   「A1_01.dwg 已提取 3 个布局：平面布置图/地面铺装图/天花布置图」
```
