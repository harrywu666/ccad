## 室内装饰施工图 AI 自动审核系统

**版本：V3.0 | 日期：2026年2月**
**注：本版本为最终定稿版，前两版因技术路线反复验证已废弃**

## 一、产品概述

### 1.1 产品是什么

一款专为室内装饰行业设计的施工图自动审核软件。设计师或审图员上传一套完整的施工图文件，系统自动完成图纸的整理、匹配、审核，最终输出一份详细的审核报告。

整个审核过程模拟真人审图员的工作方式：先看目录了解图纸全貌，再逐张读图理解内容，最后跨图比对找出问题。

### 1.2 解决什么问题

室内装饰施工图审核是一项高度重复、耗时且容易出错的工作。一套完整的施工图通常有几十到上百张图纸，审图员需要在平面图、立面图、大样图之间反复翻阅比对，常见的问题包括：

- 平面图和立面图的同一面墙尺寸标注不一致
- 图纸中的材料标注与材料表对不上
- 索引符号指向的图纸不存在或内容对不上
- 图纸目录与实际图纸数量不符

这些错误如果流入施工阶段，轻则返工，重则造成重大损失。本软件通过AI+精确数据的方式，自动完成这些重复性的比对工作。

### 1.3 目标用户

- 室内设计师（自查用）
- 设计公司审图部门
- 施工图审查机构
- 甲方工程管理部门

### 1.4 核心竞争力

开发者具有多年室内设计从业经验，深度理解图纸审核的业务逻辑，这是本产品最大的差异化优势。

## 二、技术路线说明

### 2.1 核心设计思想

系统由三条数据线并行运行，最终汇合：

```
数据线1：目录PNG  →  Kimi识别  →  图纸目录（锚点）
数据线2：PDF     →  逐页转PNG  →  Kimi理解图纸内容
数据线3：DWG     →  CAD插件   →  JSON精确数据
                                    ↓
                            三线汇合匹配
                                    ↓
                            AI跨图审核比对
                                    ↓
                            生成完整审核报告
```

### 2.2 为什么是这三条线

**为什么需要目录：**
目录是整套图纸的权威来源，作为系统的「锚点」，所有图纸都向目录对齐。AI不是自由发挥猜测图纸是什么，而是拿着目录去核对确认，准确率大幅提升。

**为什么需要PDF转PNG：**
PNG是给AI看的，AI通过图像理解图纸的空间关系和语义（这是墙体/家具/门洞）。PDF是设计师日常导出的标准格式，质量最高。

**为什么需要DWG的JSON数据：**
AI读图有分辨率限制，A1图纸细节容易看不清，数值识别置信度不够。DWG里的标注数据是精确的，通过CAD插件直接提取，数值100%准确，弥补AI视觉识别的短板。

**两者结合：**
AI负责理解语义（这个标注是主卧东侧墙体），JSON负责提供精确数值（2400mm），缺一不可。

### 2.3 经过验证被放弃的方案

| 方案 | 放弃原因 |
|------|---------|
| ezdxf解析DXF渲染图像 | 渲染质量差，立面图尤其严重 |
| ezdxf坐标映射到PNG | 坐标系不一致，部分对得上部分对不上 |
| AI直接读A1整图 | 图片压缩，AI读数置信度低 |
| PDF提取文字 | 设计师导出PDF时常将文字转几何，数据丢失 |
| AI读裁图 | 裁图后失去整体空间上下文 |
| APS云端AutoCAD | 开发复杂，用户不需要云端部署 |
| pyautocad服务器部署 | 违反AutoCAD授权条款 |

## 三、用户使用流程

### 3.1 完整操作步骤

```
第一步：新建项目
用户输入项目名称，创建一个新的审核项目

第二步：上传目录图纸
上传图纸目录页的PNG图片
→ 系统调用Kimi识别目录内容
→ 前端展示识别结果（图号、图名、版本、日期）
→ 用户校对，手动修改识别错误的地方
→ 确认目录，锁定为本项目的铁证

第三步：上传PDF图纸集
上传完整的施工图PDF（一个PDF包含所有图纸）
→ 系统流式逐页转PNG
→ 每张PNG导出后立即调用Kimi识别图名图号
→ 自动与目录匹配
→ 前端实时展示匹配进度和结果
→ 未能自动匹配的图纸，用户手动指定

第四步：上传DWG文件
上传对应的DWG文件
→ 系统调用本地CAD插件提取数据
→ 生成JSON，与对应图纸绑定
→ 前端展示数据提取结果

第五步：确认三线匹配
前端展示完整的匹配关系表：
目录条目 ←→ 图纸PNG ←→ JSON数据
用户确认无误后，点击「开始审核」

第六步：等待审核完成
系统自动执行三步核对逻辑
前端实时展示审核进度

第七步：查看审核报告
前端展示完整审核结果
可下载PDF或Excel格式的报告
```

### 3.2 项目状态流转

```
新建项目 → 目录已确认 → 图纸匹配中 → 三线匹配完成 → 审核中 → 审核完成
```

## 四、核心审核功能

> **重要：三步审核必须按顺序执行，存在强依赖关系**
> 索引核对是基础，索引关系错了，后续所有核对都没有意义

### 4.1 第一步：索引核对

**什么是索引：**
平面图或立面图上的圆形符号，上半部分是索引编号（如①），下半部分是目标图纸的图号（如A2-01）。它的作用是告诉看图的人：「这个位置的详细做法，请看A2-01这张图的①号大样」。

**核对内容：**
- 提取所有图纸中的索引符号
- 建立索引发出方和接收方的对应关系
- 检查断链：有发出但目标图纸不存在
- 检查断链：有发出但目标图纸中找不到对应编号
- 检查孤立：目标图纸中存在索引编号，但没有任何图纸指向它

**输出：**
- 完整索引对照表
- 所有断链的详细描述（哪张图的哪个索引指向了哪里，问题是什么）

### 4.2 第二步：平立面尺寸核对

**核对内容：**
- 基于索引关系，找到平面图对应的立面图
- 比对同一面墙在平面图和立面图中的尺寸标注
- 检查总尺寸与分段尺寸是否吻合

**输出（需要非常详细）：**

每一条尺寸问题需要包含：
- 涉及哪张平面图（图号+图名）
- 涉及哪张立面图（图号+图名）
- 具体位置描述（如：B-C轴/①-②轴范围内的东侧墙体）
- 平面图标注的数值
- 立面图标注的数值
- 差值
- AI生成的自然语言描述（如：「平面图A1-01中，主卧东侧墙体净宽标注为3600mm，但对应立面图A2-03中同位置标注为3400mm，差值200mm，请核实」）

### 4.3 第三步：材料核对

**核对内容：**
- 材料表的编号和名称 vs 平立面引线标注
- 图纸中出现的材料是否都在材料表中有定义
- 材料表中的材料是否都在图纸中有实际使用
- 相同编号但名称不一致的材料（AI做模糊匹配）

**输出（同样需要详细）：**

每一条材料问题包含：
- 问题类型（未定义/未使用/名称不一致）
- 涉及图纸（图号+图名）
- 具体位置
- AI生成的自然语言描述

## 五、审核报告

### 5.1 前端展示

报告分四个区域展示：

**概览区：**
- 审核项目名称和日期
- 总问题数
- 各类型问题数量（索引/尺寸/材料）
- 整体评级（通过/需整改/严重问题）

**索引问题列表：**
- 每条问题独立展示
- 点击可定位到对应图纸位置

**尺寸问题列表：**
- 每条问题包含完整的自然语言描述
- 显示涉及的图号、位置、数值差异

**材料问题列表：**
- 每条问题包含完整描述
- 显示涉及的图号和位置

### 5.2 可下载报告

**PDF报告：**
- 封面（项目名称、审核日期、问题总数）
- 问题概览表
- 各类问题详细列表（含AI自然语言描述）
- 附件：问题定位图（在图纸PNG上标出问题位置）

**Excel报告：**
- Sheet1：概览
- Sheet2：索引问题清单
- Sheet3：尺寸问题清单
- Sheet4：材料问题清单
- 每列包含：图号、图名、位置、问题描述、严重程度

## 六、产品边界（不做什么）

- 不做CAD图纸的在线编辑功能
- 不做云端DWG存储（DWG文件本地处理，不上传服务器）
- 不做设计规范检查（通道净宽、房间面积等）
- 不做自动修改图纸功能
